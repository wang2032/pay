# 算力市场轻量版平台专业需求文档 v0.1

## 1. 文档目的

本文档用于明确"算力市场轻量版平台"的专业需求，按**前端 / 后端**拆分实现范围，清晰描述：

- 每个模块要实现什么功能
- 由哪一层承担
- 第一阶段上线哪些能力
- 第二阶段及未来预留哪些能力

本文档偏"需求规格说明"，用于交付前后端研发、架构师、Codex 或项目管理人员做任务拆分与开发实施。

---

## 2. 系统角色划分

### 2.1 前端

前端基于 **Paymenter** 二开，承担：

- 市场展示
- 创建实例表单
- 服务页展示
- 费用页展示
- 操作按钮入口
- 用户可见状态与文案

前端不承担：

- 实例控制
- 计费计算
- SSH/JupyterLab 接入
- 存储管理
- 状态机执行

### 2.2 后端

后端基于现有项目能力提炼和扩展，承担：

- 资源套餐与价格方案服务
- 创建请求受理与实例控制器
- 实例状态管理
- 计费与余额服务
- SSH / JupyterLab 访问入口生成
- 异常、到期、欠费处理
- 与 Paymenter 的接口对接

---

## 3. 前端需求

# 3.1 市场列表页

## 功能目标

为用户提供资源套餐浏览、筛选、对比与进入创建流程的入口。

## 必须实现的功能

### 3.1.1 资源套餐卡片展示

每个卡片必须展示：

- GPU 型号
- GPU 数量
- 显存
- CPU 数量
- 内存
- 地区
- 支持的价格方案标签（按量/包天/包周/包月）
- 当前展示价格
- SSH / JupyterLab 标签
- 库存状态（可购买 / 紧张 / 售罄）

### 3.1.2 市场筛选

第一阶段至少支持：

- 按地区筛选
- 按 GPU 型号筛选
- 按价格排序
- 按可购买状态筛选

### 3.1.3 资源详情入口

卡片必须支持点击查看详情抽屉/详情页，展示：

- CPU 型号
- GPU 驱动版本
- CUDA 版本
- 本地存储说明
- 规则摘要
- 镜像适配说明（只读）

### 3.1.4 创建入口

每个卡片必须提供：

- "立即创建"或等效按钮

## 第一阶段不要求

- 复杂多维筛选
- 高级比较器
- 实时真实库存数字展示

## 需要调用的后端接口

- 市场列表查询
- 套餐详情查询
- 筛选项查询（可合并）

---

# 3.2 创建实例页

## 功能目标

完成一次实例创建所需的全部用户输入与展示。

## 页面结构

- 左侧配置区
- 右侧费用/规则摘要区

## 必须实现的功能

### 3.2.1 资源确认区

只读展示：

- 资源套餐名称
- GPU 型号
- GPU 数量
- 显存
- CPU 数量
- CPU 型号
- 内存
- 地区
- 驱动版本
- CUDA 版本
- 访问方式（SSH / JupyterLab）

### 3.2.2 价格方案选择

提供单选：

- 按量
- 包天
- 包周
- 包月

每个选项必须显示价格。

### 3.2.3 镜像选择

第一阶段支持平台预设镜像。

镜像项必须展示：

- 镜像名称
- 版本号
- 基础系统
- CUDA 版本
- 框架说明
- 更新时间
- 兼容标签（兼容 / 警告 / 不可用）

### 3.2.4 SSH 初始化方式

至少支持两种：

- 上传 SSH 公钥
- 输入初始化密码

前端必须对格式与必填项做基础校验。

### 3.2.5 JupyterLab 说明

创建页需明确展示：

- 创建完成后可直接打开 JupyterLab
- JupyterLab 为平台默认访问方式之一

### 3.2.6 实例名称

支持用户输入实例名称。

前端校验：

- 长度限制
- 合法字符限制

### 3.2.7 右侧费用摘要

展示：

- 当前资源套餐摘要
- 当前价格方案
- 镜像摘要
- 本地存储说明
- 计费规则摘要
- 创建中/失败是否计费说明
- 余额是否足够

### 3.2.8 提交创建

按钮文案应支持：

- 创建实例
- 余额不足时：去充值 / 充值后创建

## 第一阶段不上线的能力

- 我的镜像
- 保存镜像
- 共享存储开关
- 启动脚本模板
- 更复杂的初始化配置

## 需要调用的后端接口

- 创建页初始化数据接口
- 镜像兼容性校验接口
- 余额校验接口
- 创建实例提交接口

# 3.4 实例详情页 / 服务页

## 功能目标

集中展示实例当前状态、访问信息、计费信息和运行中操作。

## 必须实现的功能

### 3.4.1 实例状态展示

展示：

- 实例状态
- 计费状态
- 资源套餐信息
- 当前价格方案
- 镜像信息

### 3.4.2 SSH 信息展示

展示：

- SSH 主机名
- SSH 端口
- SSH 用户名
- 登录方式（密码/公钥）
- 复制 SSH 命令按钮

### 3.4.3 JupyterLab 入口

展示：

- 打开 JupyterLab 按钮
- 复制访问链接按钮（可选）

### 3.4.4 本地存储说明

展示：

- 本地存储路径
- 删除实例后数据是否保留说明

### 3.4.5 计费信息

按量实例展示：

- 小时单价
- 已运行时长
- 当前累计费用
- 当前余额
- 计费状态

包期实例展示：

- 开始时间
- 到期时间
- 剩余时间
- 是否已到期

### 3.4.6 动作按钮

第一阶段必须支持：

- 停止实例
- 启动实例
- 删除实例
- 续费（仅包期）
- 复制 SSH 命令
- 打开 JupyterLab
- 去充值（按量余额不足时）

## 第一阶段不上线

- 保存镜像
- 重置系统
- 转按量/转包期
- 文件管理器

## 需要调用的后端接口

- 实例详情查询接口
- 停止实例接口
- 启动实例接口
- 删除实例接口
- 续费接口

## 4. 后端需求

# 4.1 资源套餐与价格方案服务

## 功能目标

统一管理资源套餐与价格方案，向前端提供市场页与创建页数据。

## 必须实现的功能

### 4.1.1 资源套餐模型

每个资源套餐至少包含：

- 名称
- GPU 型号
- GPU 数量
- 显存
- CPU 数量
- CPU 型号
- 内存
- 地区
- 驱动版本
- CUDA 版本
- 本地存储容量
- 是否支持 SSH
- 是否支持 JupyterLab
- 是否启用

### 4.1.2 价格方案模型

每个价格方案至少包含：

- 方案类型（按量/包天/包周/包月）
- 单价
- 币种
- 是否启用
- 生效时间

### 4.1.3 套餐与方案关联

- 一个资源套餐可关联多个价格方案
- 前端按套餐 + 方案进行展示

### 4.1.4 市场状态计算

计算并输出：

- 可购买
- 紧张
- 售罄

## 可复用现有项目能力

- 若已有资源池查询能力，可复用库存聚合逻辑
- 若已有套餐配置能力，可复用配置存储与后台管理能力

---

# 4.2 创建受理与实例控制服务

## 功能目标

受理前端创建请求，并异步完成实例创建。

## 必须实现的功能

### 4.2.1 提交前校验

必须校验：

- 用户身份
- 资源套餐有效性
- 价格方案有效性
- 镜像兼容性
- SSH 公钥/密码合法性
- 余额/支付条件

### 4.2.2 创建受理

- 创建实例记录
- 创建任务记录
- 返回"已受理"状态

### 4.2.3 异步创建流程

内部至少包含：

- 资源再次校验
- 实例资源创建
- 容器启动
- SSH 初始化
- JupyterLab 初始化
- 访问入口分配
- 成功标记为可用
- 失败标记为失败

### 4.2.4 取消创建

- 创建中可取消
- 取消后清理已创建资源
- 不进入计费

### 4.2.5 超时处理

- 创建超时自动失败
- 写入错误原因

## 可复用现有项目能力

- 如已有 K8s 资源创建逻辑，可复用资源控制/任务执行框架
- 如已有异步任务系统，可复用任务调度与重试机制

---

# 4.3 访问接入服务

## 功能目标

为每个实例生成 SSH 与 JupyterLab 的可访问入口。

## 必须实现的功能

### 4.3.1 SSH 接入

- 分配 SSH 外部访问入口
- 生成 SSH 主机名、端口、用户名
- 支持密码或公钥登录模式
- 回写前端展示信息

### 4.3.2 JupyterLab 接入

- 准备 JupyterLab 服务
- 生成一键访问 URL
- 保证前端展示可直接点击

### 4.3.3 可用性检查

- SSH 健康检查
- JupyterLab 健康检查
- 两者均通过后才标记可用

## 可复用现有项目能力

- 若已有统一网关、域名分发或反向代理能力，可复用
- 若已有认证/令牌分发能力，可复用到 JupyterLab 入口

---

# 4.4 计费与余额服务

## 功能目标

实现按量运行态计费、包期计时、余额扣减、欠费停机等逻辑。

## 必须实现的功能

### 4.4.1 按量计费

- 创建时校验最低启动门槛
- 实例真正可用后写入 billing_started_at
- 周期性累积使用时长
- 扣减余额
- 停止/删除/异常/欠费后停止计费

### 4.4.2 包期计费

- 支付成功后写入 period_started_at
- 记录到期时间
- 支持续费延长 period_end_at
- 停机不停止计时

### 4.4.3 状态分离

必须分离：

- 实例状态
- 计费状态

### 4.4.4 欠费处理

- 余额不足时进入待停机/预警
- 短缓冲期后停机
- 不立即删除实例

### 4.4.5 失败与补偿

- 创建中/失败/取消不计费
- 异常订单需支持作废/补偿/人工核对

## 可复用现有项目能力

- 若已有账本、钱包、预充值、订单系统，可复用余额和流水能力
- 若已有定时任务系统，可复用按量扣费与到期巡检逻辑

---

# 4.5 实例生命周期管理服务

## 功能目标

实现实例运行中的控制动作与状态流转。

## 必须实现的功能

### 4.5.1 停止实例

- Running -> Stopping -> Stopped
- 按量实例停止计费
- 保留本地数据

### 4.5.2 启动实例

- Stopped -> Starting -> Running / Failed
- 恢复实例访问
- 按量实例恢复计费

### 4.5.3 删除实例

- Running/Stopped/Error -> Terminating -> Terminated
- 清理本地数据
- 停止计费
- 保留记录

### 4.5.4 续费

- 仅包期实例支持
- 支付成功后延长到期时间

### 4.5.5 到期处理

- 包期到期后默认进入 Expired
- 停止服务
- 不自动转按量

### 4.5.6 异常处理

- 进入 Error
- 保留错误原因
- 支持人工补偿或用户删除

## 可复用现有项目能力

- 若已有实例状态巡检能力，可复用状态同步与异常检测
- 若已有回收任务系统，可复用删除/回收执行器

---

# 4.6 存储服务（第一阶段预留）

## 功能目标

第一阶段仅对外提供本地存储；共享存储能力后端预留。

## 第一阶段必须实现

### 本地存储

- 本地盘容量随资源套餐固定
- 停止保留、删除清理
- 固定路径输出给前端

## 第一阶段预留但不对外开放

### 共享存储

- 用户级共享目录模型
- 挂载点字段
- 生命周期字段
- 创建流程中的可选挂载位

## 可复用现有项目能力

- 若已有 PVC/卷管理能力，可复用本地盘管理
- 若已有网络存储基础设施，可为二阶段共享存储预留接口

# 4.7 与 Paymenter 对接服务

## 功能目标

通过 Paymenter 承接市场、订单、支付与服务页，后端提供数据与动作能力。

## 必须实现的功能

### 4.7.1 市场与详情数据提供

向前端/Paymenter 页面提供：

- 市场卡片数据
- 套餐详情数据
- 价格方案数据
- 规则摘要数据

### 4.7.2 创建与服务页联动

- 创建实例接口
- 实例详情接口
- 操作接口（停止/启动/删除/续费）
- 服务页状态回填

### 4.7.3 服务动作支持

需支持前端展示并调用：

- 复制 SSH 命令
- 打开 JupyterLab
- 停止/启动/删除实例
- 续费
- 去充值

### 4.7.4 回调/同步

- 创建状态同步
- 可用状态同步
- 失败状态同步
- 费用摘要同步

## 5. 状态机需求

# 5.1 实例状态

至少支持：

- Pending
- Creating
- Running
- Stopped
- Failed
- Cancelled
- Expired
- Suspended
- Error
- Terminated

# 5.2 计费状态

至少支持：

- NotStarted
- BillingActive
- BillingStopped
- PeriodEnded
- PaymentInsufficient
- Closed

# 5.3 关键规则

- 实例状态与计费状态分离
- 订单支付成功 != 实例创建成功
- 只有 SSH + JupyterLab 均可用才算 Running
- 停止 != 删除
- 到期 != 删除

## 8. 结论

本项目第一阶段的研发重点，不是"做一个很大的平台"，而是把以下闭环做稳：

**看得懂资源 → 选得对价格 → 创建得出来 → 进得去实例 → 算得清费用 → 管得住生命周期**

Paymenter 负责前台壳与服务页，真正的产品壁垒在后端：

- 实例控制
- 访问入口
- 计费规则
- 到期与欠费处理
- 后续镜像与存储扩展能力
